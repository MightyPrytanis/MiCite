Security Blueprint

MiCite is designed around deterministic, no-retention citation-formatting review. In the default checking workflow, document text is processed in your browser. MiCite has no account system, no database for user documents, no third-party scripts, and no remote AI model call. Everything is processed locally except optional CourtListener parallel-citation lookup. If the user expressly enables that lookup, the only information exposed outside the browser for that feature is the extracted case-citation identifier needed for lookup: volume, reporter, and page. MiCite does not send the full document, argument, factual discussion, party discussion, notes, citation tables, corrected output, or surrounding context for that lookup. Text is checked in memory inside the page v [Report / Corrected Text / Formatted HTML] | | 3. Output remains in the browser unless the user copies or downloads it v [User] Optional parallel-citation lookup flow: [User Browser] | | 1. Volume/reporter/page lookup request only v [CourtListener Citation Lookup API] | | 4. Matching public reporter citations returned v [MiCite Browser Output]

Privacy Policy

Effective date: June 20, 2026.

This Privacy Policy describes how MiCite handles user text, files, citation lookups, and ordinary web request metadata. For ownership, license, warranty, no-legal-advice, and no-endorsement terms, please see the Terms of Use.

MiCite does not require user accounts, logins, passwords, billing information, or personally identifying information.

MiCite does not use cookies, advertising pixels, behavioral trackers, third-party analytics scripts, or cross-site tracking technologies. MiCite does not harvest, retain, sell, rent, or share document contents, citation reports, corrected text, formatted output, or user-supplied legal materials.

MiCite does not send uploaded TXT/MD files, full document text, citation reports, corrected text, formatted output, legal argument, notes, or surrounding citation context to an LLM provider, remote database, or remote formatting server. The deployed formatting engine performs citation-formatting review in the browser.

Everything is processed locally except optional CourtListener parallel-citation lookup. If the user opts into that lookup, the only information exposed outside the browser for that feature is the extracted case-citation identifier needed for lookup: volume, reporter, and page. The CourtListener lookup is used only to retrieve possible public parallel citations. Users should independently verify all returned citations before filing.

Vercel or other hosting infrastructure may process ordinary web request metadata needed to serve static files, protect the service, diagnose errors, and understand aggregate usage, such as IP address, user-agent, timestamp, requested URL, and response status. MiCite is designed so document text and surrounding legal context are not included in those requests, and Cognisint does not sell or rent that request metadata.

Contact: info@cognisint.com.

MiCite checks citation format only. It does not validate the existence of a cited case or source, verify quotations or legal inferences, or determine whether an authority has been overruled, superseded, or otherwise limited. MiCite is a formatting tool, not a cite-checking or legal-research tool.
